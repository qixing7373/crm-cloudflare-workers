# 旧方案存档 — 资源池分配模式

> **状态**：❌ 已废弃，仅供参考  
> **废弃日期**：2026-04-03  
> **替代方案**：产品需求文档-v3.x（简化版：总库去重 + 转化标记）  
> **废弃原因**：甲方确认实际业务不需要资源池分配、脱敏、申请审批等复杂流程，改用更简单的总库去重模式

---

## 一、方案概述

本方案采用「**资源池**」机制管理客户数据：所有客户数据统一存放在资源池中，
普通用户无法直接查看，必须通过「申请 → 审批 → 分配」流程获得限量、限时的数据访问权限。

**核心理念**：客户数据是一次性资源，被查看过就永久消费，未查看的过期后回收复用。

## 二、角色与权限

| 角色 | 录入客户 | 客户列表 | 客户详情 | 编辑客户 | 审批申请 |
|------|:-------:|:-------:|:-------:|:-------:|:-------:|
| **超级管理员** | ✅ | ✅ 全量，不脱敏 | ✅ 直接查看，不计次 | ✅ | ✅ |
| **管理员** | ✅ | ✅ 全量，不脱敏 | ✅ 直接查看，不计次 | ✅ | ✅ |
| **普通用户** | ✅ 录完即入池 | ⚠️ 仅已分配 + 脱敏 | ⚠️ 消耗查看次数 | ❌ 只读 | ❌ |

## 三、核心流程

### 流程 A：录入客户

- 任何用户可录入客户（必填：名称、电话）
- 录入时自动查重（硬查重：手机号；软查重：名称+公司）
- 录入后客户进入资源池，录入人自己也看不到

### 流程 B：申请查看

员工申请时填写：筛选条件 + 申请条数 + 天数 + 每条查看次数 + 理由

### 流程 C：管理员审批

- 通过 → 系统自动从资源池随机分配指定数量客户（排他分配，使用行锁防并发）
- 驳回 → 填写理由通知申请人
- 资源不足时阻止通过

### 流程 D：查看数据

- 普通用户列表：仅已分配且未过期的客户，全部脱敏显示
- 查看详情：消耗一次查看次数，显示完整数据（不脱敏）
- 5分钟内重复查看不重复计次
- 管理员列表：全量数据，不脱敏，不计次
  - Tab：全部 / 资源池 / 已分配 / 已消费

### 流程 E：追加查看次数

查看次数用完后，可申请追加次数，管理员审批通过后追加

### 流程 F：过期与回收

- 已查看的（view_count ≥ 1）→ 永久消费，不再分配
- 未查看的（view_count = 0）→ 回归资源池，可被再次申请
- cron 每小时扫描 + 查询时实时过滤，双保险

### 流程 G：管理员直接分发

管理员可从资源池勾选客户直接分发给指定用户，无需申请审批

## 五、数据表设计

| 表名 | 用途 |
|------|------|
| crm_customer | 客户资料（名称、电话、来源、录入人、状态） |
| crm_tag | 标签（名称、颜色） |
| crm_customer_tag | 客户-标签关联 |
| crm_remark | 客户备注（内容、附件、记录人、不可删除） |
| crm_access_request | 查看申请（条数、天数、次数、筛选JSON、类型：新申请/追加） |
| crm_customer_assign | 客户分配记录（最大查看次数、已查看次数、过期时间、状态） |
| crm_view_log | 查看日志（客户ID、查看人、时间） |
| crm_import_batch | 导入批次记录 |

## 六、资源池状态流转

```
                    ┌─────────┐
                    │ 资源池   │ ← 所有未被占用的客户
                    │（可分配）│
                    └────┬────┘
                         │ 审批通过 / 管理员直接分发
                         ▼
                    ┌─────────┐
                    │ 已分配   │ ← 锁定中，不可再分
                    │（生效中）│
                    └────┬────┘
                         │ 过期
              ┌──────────┴──────────┐
              ▼                     ▼
        ┌──────────┐          ┌──────────┐
        │ 已消费    │          │ 已回收    │
        │view≥1    │          │view=0    │
        │永久锁定  │          │回归资源池  │
        └──────────┘          └──────────┘
```

## 七、与现行方案的对比

| 项目 | 本方案（资源池模式） | 现行方案（总库去重模式） |
|------|:------------------:|:--------------------:|
| 核心概念 | 资源池 + 申请审批 + 分配 | 总库 + 去重 + 转化标记 |
| 业务员操作 | 申请 → 审批 → 脱敏列表 → 消耗次数查看 | 输入号码 → 查询 → 一键录入转化 |
| 数据模型 | 多维客户资料（名称/电话/公司/行业/地区） | 仅手机号 |
| 去重方式 | 排他分配（同一时间只一人） | 90天时效去重 + 已转化永久去重 |
| 复杂度 | 高（8张表、20+接口） | 低（3张表、8个接口） |
