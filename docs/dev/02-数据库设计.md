## 数据模型总览

### 1. 系统与组织基建 (System & Org)
| 表名 | 中文说明 | 核心用途与设计意图 |
|:---|:---|:---|
| **`contact_field`** | 联系人动态字段 | 控制联系人主表的弹性结构数据，支持新增/修改表单类型。 |
| **`user_group`** | 用户分组 (Group) | 用于人员分组，实现"管理只能看本分组数据"的权限隔离要求。 |
| **`user`** | 员工账户 | 超管/分组主管/业务员的账号信息、密码及登录状态控制底座。 |

### 2. 生成邀请与安全通道 (已重构)
*邀请码机制已升级为 O(1) 的 `HMAC(secret, user_id + 日期)` + XOR 掩码动态推导计算，**`invitation_code` 表已被永久移除**。每日更换，无存储负担。*

### 3. 联系人核心资产 (Contact Assets)
| 表名 | 中文说明 | 核心用途与设计意图 |
|:---|:---|:---|
| **`contact`** | 联系人主记录 | 核心主表，统一记录脱敏流转状态和私海所有权 (owner_id)。 |

### 4. 数据流转与操作审计 (Data Flow & Audit)
| 表名 | 中文说明 | 核心用途与设计意图 |
|:---|:---|:---|
| **`import_log`** | 导入日志 | 汇总单次 Excel 导入中"去重阻挡/覆盖/全新"的数据报告。 |
| **`contact_log`** | 联系人变更日志 | 局部行为日志，生成详情页的"联系人时间轴"（记录 Diff）。 |
| **`user_log`** | 全局安全审计 | 大一统日志，记录员工的登录、脱敏搜索追踪、禁用及信息修改。 |

> 自增ID主键。手机号可为空，有手机号时用于去重。`owner_id` 标识私海归属人。

| **`first_imported_at`** | `timestamp` | - | - | 首次入库时间 |
| **`latest_imported_at`** | `timestamp` | - | - | 最近导入时间 |
| **`created_by`** | `integer` | - | - | 创建人 |
| **`created_at`** | `timestamp` | - | `now()` | 创建时间 |
| **`updated_by`** | `integer` | - | - | 最后修改人 |
| **`updated_at`** | `timestamp` | - | - | 最后修改时间 |
| **`deleted_at`** | `timestamp` | - | - | 软删除时间 |

```typescript
export const contact = sqliteTable('contact', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  phone: text('phone').unique(),                 // 去重用 (可为空)

  owner_id: integer('owner_id'),                 // developed 时绑定归属权 (FK → user.id)
  claimed_at: integer('claimed_at', { mode: 'timestamp' }), // 私域保护的倒计时起点
  import_count: integer('import_count').default(0),          // 累计被导入次数
  first_imported_at: integer('first_imported_at', { mode: 'timestamp' }),
  latest_imported_at: integer('latest_imported_at', { mode: 'timestamp' }),
  created_by: integer('created_by'),
  created_at: integer('created_at', { mode: 'timestamp' }).default(sql`(unixepoch())`),
  updated_by: integer('updated_by'),
  updated_at: integer('updated_at', { mode: 'timestamp' }),
  deleted_at: integer('deleted_at', { mode: 'timestamp' }),
})
// 索引: phone (唯一，允许 null), status, owner_id, claimed_at
```

data 示例：
```json
// 有手机号
{ "phone": "+66812345678", "name": "Wang Wu",
  "gender": "mr", "age": 28, "country": "Thailand", "province": "Bangkok",
  "city": "Sukhumvit", "wechat": "wangwu88", "source": "douyin", "remark": "" }
```

> **跌落机制风控说明**：若 `status` == `developed`，必须在业务层设立每日运行的高可用调度任务（Cron Job/定时 Worker），当 `(Now - claimed_at) > 90天`，强制将其重置为 `undeveloped`，`owner_id` 及 `claimed_at` 设为空，使其重新回流至系统公海，并向 `contact_log` 追加释放留痕。

## 导入核心逻辑

```typescript
async function runImport(row_list: ImportRow[], user_id: number, import_id: number, db: DrizzleD1) {
  const _stats = { added: 0, updated: 0, reimported: 0, skipped: 0, frozen: 0 }
  const _now = Math.floor(Date.now() / 1000)

  for (const _row of row_list) {
    const _phone = _row.data.phone || null

    // 1. 无手机号 → 直接新建（不参与去重）
    if (!_phone) {
      const _inserted = await db.insert(contact).values({
        phone: null, data: JSON.stringify(_row.data),
      }).returning({ id: contact.id })

      await db.insert(contactChangeLog).values({
        contact_id: _inserted[0].id,
        user_id: user_id, import_id,
        type: 'create', changes: null,
      })
      _stats.added++
      continue
    }

    // 2. 有手机号 → 按 phone 查已有记录
    const _existing = await db.select().from(contact)
      .where(eq(contact.phone, _phone)).get()

    // 检测是否受高优权限保护：唯一准则——是否已处于【已开发 (developed)】
    const _is_developed = _existing && _existing.status === 'developed'

    if (_is_developed) {
      // 触发永久保护机制 → 冻结主记录阻止覆盖，仅底层记录导入企图
      const _old_data = JSON.parse(_existing.data)
      const _changes = diffRecord(_old_data, _row.data)
      await db.insert(contactChangeLog).values({
        contact_id: _existing.id,
        user_id: user_id, import_id,
        type: 'frozen_import',
        changes: Object.keys(_changes).length > 0 ? JSON.stringify(_changes) : null,
      })
      _stats.frozen++
      continue
    }

    if (_existing) {
      const _old_data = JSON.parse(_existing.data)
      const _new_data = _row.data
      const _changes = diffRecord(_old_data, _new_data)
      const _has_change = Object.keys(_changes).length > 0

      // 同一人导入 + 数据无变化 → 拒绝
      const _last_log = await db.select().from(contactChangeLog)
        .where(and(
          eq(contactChangeLog.contact_id, _existing.id),
          eq(contactChangeLog.user_id, user_id),
        ))
        .orderBy(desc(contactChangeLog.created_at))
        .limit(1).get()

      if (_last_log && !_has_change) {
        _stats.skipped++
        continue
      }

      if (_has_change) {
        await db.update(contact).set({
          data: JSON.stringify(_new_data),
          latest_imported_at: _now,
          import_count: _existing.import_count + 1,
        }).where(eq(contact.id, _existing.id))

        await db.insert(contactChangeLog).values({
          contact_id: _existing.id,
          user_id: user_id, import_id,
          type: 'update', changes: JSON.stringify(_changes),
        })
        _stats.updated++
      } else {
        // 不同人导入 + 无变化 → 记录导入事件
        await db.update(contact).set({
          latest_imported_at: _now,
          import_count: _existing.import_count + 1,
        }).where(eq(contact.id, _existing.id))

        await db.insert(contactChangeLog).values({
          contact_id: _existing.id,
          user_id: user_id, import_id,
          type: 'reimport', changes: null,
        })
        _stats.reimported++
      }
    } else {
      // phone 不存在 → 新建
      const _inserted = await db.insert(contact).values({
        phone: _phone, data: JSON.stringify(_row.data),
        first_imported_at: _now, latest_imported_at: _now,
      }).returning({ id: contact.id })

      await db.insert(contactChangeLog).values({
        contact_id: _inserted[0].id,
        user_id: user_id, import_id,
        type: 'create', changes: null,
      })
      _stats.added++
    }
  }
  return _stats
}

// 计算两个 data JSON 对象的差异
function diffRecord(old_data: Record<string, any>, new_data: Record<string, any>) {
  const _changes: Record<string, { old: any; new: any }> = {}
  const _all_keys = new Set([...Object.keys(old_data), ...Object.keys(new_data)])
  for (const _key of _all_keys) {
    if (old_data[_key] !== new_data[_key]) {
      _changes[_key] = { old: old_data[_key] ?? null, new: new_data[_key] ?? null }
    }
  }
  return _changes
}
```

> **批量插入**：D1 单次事务最大 1000 行，超过时分批处理。
