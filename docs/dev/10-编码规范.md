> 代码洁癖级别的命名标准与开发规范

## 极客编程理念

> **核心目标**：代码应该像一把装配精良的枪——零件少、扣扳机就响、拆卸简单。

### 1. Guard Clause（卫语句优先，消灭嵌套）

```typescript
// ❌ 箭头地狱
async function claimContact(user_id: number, contact_id: number) {
  const _contact = await findContact(contact_id)
  if (_contact) {
    if (_contact.status === 'undeveloped') {
      if (!_contact.user_id) {
        await db.update(contact).set({ status: 'developed', user_id })
        return { ok: true }
      } else {
        throw new AppError(ErrorCode.COLLISION_DETECTED)
      }
    } else {
      throw new AppError(ErrorCode.CONTACT_PROTECTED)
    }
  } else {
    throw new AppError(ErrorCode.PHONE_NOT_FOUND)
  }
}

// ✅ 卫语句：先排除异常，最后一行才是正事
async function claimContact(user_id: number, contact_id: number) {
  const _contact = await findContact(contact_id)
  if (!_contact) throw new AppError(ErrorCode.PHONE_NOT_FOUND)
  if (_contact.status === 'developed') throw new AppError(ErrorCode.CONTACT_PROTECTED)
  if (_contact.user_id) throw new AppError(ErrorCode.COLLISION_DETECTED)

  await db.update(contact).set({ status: 'developed', user_id, claimed_at: Date.now() })
}
```

### 3. 现代语法与框架糖 (Modern Syntax & Framework Features)

> **原则**：尽可能使用 TypeScript 语法糖、Hono 的自带特性、甚至位运算与解构等技巧来极限压缩代码体积和认知复杂度。

```typescript
// ❌ 传统写法（又臭又长）
const idStr = c.req.param('id');
const id = parseInt(idStr, 10);
const body = await c.req.json();
const name = body.name;
const status = body.status ? body.status : 'undeveloped';

// ✅ 语法糖 + 框架特性（短小精悍）
const id = +c.req.param('id') // 一元加号强转
const { name, status = 'undeveloped' } = await c.req.json() // 解构+默认值
```
- **Hono 特性**：利用 `c.req.query()` 默认备用值 `c.req.query('page') || '1'`，熟练运用中间件传递状态。
- **TypeScript 糖**：广泛使用 `?.` (可选链)、`??` (空值合并)、`!!` (布尔转)、`+` (数字强转) 等极致特性。
- **解构精简**：多利用对象和数组解构，特别是深度嵌套的参数。

### 4. Early Return（及早返回，减少认知负担）

```typescript
// ❌ 嵌套 if-else
function checkRole(user: UserInfo, required: string): boolean {
  if (user) {
    if (ROLE_LEVEL[user.role] >= ROLE_LEVEL[required]) {
      return true
    } else {
      return false
    }
  }
  return false
}

// ✅ 一行解决
function checkRole(user: UserInfo, required: string): boolean {
  return !!user && (ROLE_LEVEL[user.role] ?? 0) >= (ROLE_LEVEL[required] ?? 99)
}
```

### 6. 其他铁律

| 原则 | 说明 |
|------|------|
| **函数体 ≤ 30 行** | 超过就拆，函数是最小可测试单元 |
| **嵌套 ≤ 2 层** | 超过 2 层 `if/for` 必须用卫语句或提取子函数 |
| **单一职责** | 一个函数只做一件事，名字就是文档 |
| **不信任外部输入** | Zod schema 校验放在路由入口，业务层只处理干净数据 |
| **可选链 + 空值合并** | `user?.role ?? 'staff'` 代替三段 if 判断 |
| **`const` 优先** | 除循环计数器外禁止 `let`，强制不可变思维 |
| **禁止 `else`**（尽量） | 用 early return 消灭 else 分支 |
| **禁止内联样式** | `style="..."` 和 `:style="{}"` 一律禁止，使用 Tailwind class 或 `<style scoped>` |

## 第一原则：选词优先级

```
1️⃣ 标准开发用语 + 标准缩写（优先）→ parse, diff, fetch, hash, config, auth, init, token, async
2️⃣ 简单英语单词（次选）        →  load, save, find, run, make, check, list, sort
3️⃣ 绝对禁止（自创缩写）        →  usr, btn, msg, cfg, chk, phn, dedup
```

## 禁止事项

- ❌ **禁止自创缩写**：`usr` → `user`，`btn` → `button`，`msg` → `message`
- ❌ **禁止单字母变量**：`i` → `_index`，`e` → `_error`，`r` → `_result`
- ❌ **禁止无意义命名**：`data1`，`temp`，`foo`，`bar`，`xxx`
- ❌ **禁止魔法数字**：`86400` → `SECONDS_PER_DAY`，`1000` → `BATCH_SIZE`
- ❌ **禁止嵌套三元**：`a ? b ? c : d : e` → 用 `if/else`
- ❌ **禁止超长行**：单行不超过 100 字符
- ❌ **禁止 `any` 类型**：必须明确定义类型

```vue
  'pageChanged': [page: number]
}>()

const table_data   = ref<FieldRow[]>([])
const is_loading   = ref(false)

const has_data  = computed(() => table_data.value.length > 0)
const page_size = computed(() => Math.ceil(total_count.value / 10))

// ═══ 5. 方法（动词+名词）═══
async function loadField() {
  const _result = await api.get('/field')
  table_data.value = _result.data
}

async function saveField() {
  const _payload = { ...form_data }
  const _result  = await api.post('/field', _payload)
  emit('fieldSaved', _result.id)
}

// ═══ 6. 生命周期 ═══
onMounted(() => loadField())

// ═══ 7. 监听器 ═══
watch(
  () => props.field_id,
  (_new_id) => loadField(),
)
</script>
```

## 注释规范

> **原则**：注释写「为什么」，不写「做了什么」。好代码不需要解释做了什么，但必须解释为什么这样做。

### 文件头注释（必须）

每个 `.ts` 文件顶部必须包含文件头注释，说明文件职责和核心逻辑：

```typescript

 * @file   auth.ts — 认证路由
 * @desc   处理用户注册、登录、token 校验

 *         POST /auth/login    — 用户登录（签发 JWT，写审计日志）
 *         GET  /auth/check    — 验证当前 token 有效性
 * @deps   bcryptjs（密码哈希），Web Crypto API（JWT 签发）
```

### 安全注释（@security 标签）

涉及权限、加密、防注入的代码**必须**添加安全注释：

```typescript
// @security 乐观锁防撞单：WHERE status='undeveloped' 确保并发下只有一个人能认领成功
const _updated = await db.update(contact)
  .set({ status: 'developed', user_id: viewer.id })
  .where(and(eq(contact.id, _id), eq(contact.status, 'undeveloped')))

// @security 安全词校验：超管撤销操作必须输入「确认撤销」才能执行
if (_parsed.data.confirm_word !== '确认撤销') {
  return fail(c, ErrorCode.CONFIRM_MISMATCH)
}

// @security 禁用联动：禁用用户后自动释放其所有私海资源，防止数据锁死
await db.update(contact)
  .set({ status: 'undeveloped', user_id: null })
  .where(and(eq(contact.user_id, _id), eq(contact.status, 'developed')))

// @security bcrypt cost=10：密码哈希强度，Workers 环境下 ~100ms，可接受
const _hashed = await hash(plain, 10)

// @security JWT 签名使用 HMAC-SHA256（Web Crypto API），不使用 none 算法
// 验签时如果签名不匹配，crypto.subtle.verify 返回 false，直接拒绝
```
