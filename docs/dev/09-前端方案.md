# 前端方案

## 技术栈

- Vue 3。
- Naive UI。
- Pinia + `pinia-plugin-persistedstate`。
- Vue Router。
- Rsbuild。
- Axios。
- ExcelJS。
- `@sqlite.org/sqlite-wasm` + Comlink Worker。

当前项目不使用 Ant Design Vue；旧文档中的 AntD 描述已废弃。

## 目录结构

```text
web/src
├── api/              # 前端接口封装
├── components/       # 通用组件
├── plugins/          # axios、本地 SQLite、同步引擎
├── stores/           # Pinia 状态
├── views/            # 页面
├── vue-router.ts     # 路由和权限守卫
├── vue-pinia.ts      # Pinia 初始化
└── vue-i18n.ts       # i18n 初始化
```

## 路由和权限

| 路由 | 页面 | 最低角色 |
| --- | --- | --- |
| `/search` | 客户搜索 | staff |
| `/database` | 客户跟进 | staff |
| `/stats` | 个人统计 | staff |
| `/teamstats` | 团队统计 | manager |
| `/import` | 数据导入 | manager |
| `/users` | 用户列表 | manager |
| `/logs` | 系统日志 | manager |
| `/overview` | 工作台概览 | superadmin |
| `/groups` | 组织管理 | superadmin |
| `/fields` | 字段配置 | superadmin |

权限守卫在 `web/src/vue-router.ts` 中维护，菜单根据 `user().role_level` 动态展示。

## API 封装

所有接口封装放在 `web/src/api/*`：

- `auth.ts`：登录、注册、校验、邀请码、修改密码。
- `contact.ts`：搜索、认领、列表、详情、撤销、转移、删除、更新。
- `field.ts`：动态字段。
- `group.ts`：组织。
- `user.ts`：用户。
- `import.ts`：Hash 校验、分片同步、导入历史、明细。
- `stat.ts`：个人、团队、全局概览统计。
- `log.ts`：用户日志和联系人日志。

Axios 拦截器负责：

- 自动注入 JWT。
- 深度 trim 请求参数。
- 统一处理 `code < 0`。
- Token 过期后退出登录。

## 批量导入前端流程

导入页面不直接上传原始文件到后端：

1. 浏览器读取文件。
2. 前端解析和清洗记录。
3. 写入本地 SQLite 队列。
4. 调用 `/api/import/verify-hash` 检查重复。
5. 调用 `/api/import/sync` 分片同步。
6. 同步完成后刷新云端导入历史。

前端应保留本地任务状态，避免大文件上传过程中页面刷新或网络抖动导致全部重来。

## 构建

```bash
cd web
bun install
bunx tsc --noEmit --pretty false
bun run build
```

开发代理：

- `/api` -> `http://127.0.0.1:8888`
- `/auth` -> `http://127.0.0.1:8888`

生产环境通过 `PUBLIC_API_URL` 指向 Workers API。
