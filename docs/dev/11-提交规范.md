### Git 提交规范 (Commit Convention)
所有项目的历史和未来提交必须严格遵循以下**英文前缀 + 中文说明**的标准格式（Conventional Commits），严禁混用全中文前缀（如“重构：”、“新增：”等）或中英文符号混用：
- `feat(scope?): [功能中文描述]` - 新增功能 (feature)
- `fix(scope?): [修复中文描述]` - 修复 Bug (bugfix) 
- `docs(scope?): [文档中文描述]` - 文档更新 (documentation)
- `style(scope?): [样式中文描述]` - 代码格式/规范 (formatting, missing semi colons, etc.)
- `refactor(scope?): [重构中文描述]` - 代码重构 (refactoring)
- `perf(scope?): [性能优化中文描述]` - 性能优化 (performance)
- `test(scope?): [测试中文描述]` - 测试用例增删改 (testing)
- `chore(scope?): [杂项中文描述]` - 构建过程、辅助工具、依赖的杂项更改 (chore)
示例：`feat(api): 构建邀请码系统` 或 `refactor(web): 清理冗余的依赖导入`
