# 粉库系统 — 测试方案

> Vitest + Hono test client + Playwright

---

## 测试工具

| 层级 | 工具 | 说明 |
|------|------|------|
| 单元测试 | **Vitest** | 与 Vite 生态一致，支持 TypeScript |
| API 测试 | **Vitest + Hono test client** | Hono 内置 `app.request()` 直接测试路由 |
| E2E 测试 | **Playwright** | 浏览器端到端测试 |
| D1 测试 | **Miniflare** | Cloudflare 本地模拟器，内含 D1 SQLite |
| 安全测试 | **Vitest + 手动渗透** | JWT 篡改、越权、注入等 |

## A. 单元测试（核心逻辑）

### 去重逻辑

| # | 测试用例 | 输入 | 预期结果 |
|---|---------|------|---------| 
| 1 | 全新号码 | 总库为空，上传 `[A, B, C]` | new 3 |
| 2 | 完全重复 | 总库有 `[A]` (undeveloped)，不同人上传 `[A]`（无变化） | reimport 1（仅记录导入事件） |
| 3 | 数据更新 | 总库有 `[A]` (undeveloped)，上传 `[A]`（数据变了） | update 1（更新主记录+记录diff） |
| 4 | 已开发防写 | 已开发 (developed) 有 `[A]`，上传 `[A]` | frozen 1（触发数据保护，冻结不覆盖，记录企图） |
| 5 | 上传内去重 | 上传 `[A, A, A]` | new 1（文件内去重，保留最后一条数据容错） |
| 6 | 混合场景 | 无变化的 `[A]`，已开发的 `[B]`，新数据 `[C]` | new 1(C), reimport 1(A), frozen 1(B) |
| 8 | 空列表 | 上传 `[]` | 0，无错误 |

```typescript
describe('findDupe', () => {
  it('主记录数据变化应标记为 update', async () => {
    // seed: 插入 A (job=教师)
    const _result = await findDupe([{ phone: 'A', data: { job: '司机' } }], db, user1)
    expect(_result.update_list).toHaveLength(1)
    expect(_result.update_list[0].changes.job).toEqual({ old: '教师', new: '司机' })
  })

  it('数据无变化应拦截并标记为 reimport', async () => {
    // seed: 插入 A (job=教师)
    const _result = await findDupe([{ phone: 'A', data: { job: '教师' } }], db, user2)
    expect(_result.reimport_list).toHaveLength(1)
  })

  it('属于已开发状态的资源导入应触发防写冻结', async () => {
    // seed: 插入 A (拥有者=user2, 状态=developed)
    const _result = await findDupe([{ phone: 'A', data: { job: '主管' } }], db, user1)
    // 保护开启：进入 frozen_list 且不会覆盖主表
    expect(_result.frozen_list).toHaveLength(1)
  })
})

## D. 安全渗透测试

> 模拟黑客攻击，验证系统防护能力

### D1. JWT 攻击

| # | 攻击方式 | 测试方法 | 预期结果 |
|---|---------|---------|---------|
| 1 | **JWT 篡改** | 修改 payload 中的 role 为 superadmin | 401（签名验证失败） |
| 2 | **JWT 算法替换** | 将 header 中 alg 改为 `none` | 401（拒绝 none 算法） |
| 3 | **JWT 过期重放** | 使用已过期的 token 发请求 | 401 `TOKEN_EXPIRED` |
| 4 | **JWT 空签名** | 发送无签名的 JWT | 401 |
| 5 | **JWT 弱密钥** | 用常见弱密钥尝试签发 token | 401（密钥 ≥ 32字符） |

```typescript
describe('JWT 安全', () => {
  it('篡改 role 应被拒绝', async () => {
    const _fake_token = signJwt({ id: 1, role: 'superadmin' }, 'wrong-secret')
    const _response = await app.request('/api/field', {
      headers: { Authorization: `Bearer ${_fake_token}` },
    })
    const _json = await _response.json() as any
    expect(_json.code).toBe(-401)
  })

  it('alg=none 应被拒绝', async () => {
    const _none_token = makeNoneAlgToken({ id: 1, role: 'superadmin' })
    const _response = await app.request('/api/field', {
      headers: { Authorization: `Bearer ${_none_token}` },
    })
    const _json = await _response.json() as any
    expect(_json.code).toBe(-401)
  })
})
```

### D2. 越权攻击（IDOR / 垂直越权）

| # | 攻击方式 | 测试方法 | 预期结果 |
|---|---------|---------|---------|
| 1 | **垂直越权** | 业务员 token 访问管理员 API | code `-400` |
| 2 | **垂直越权** | 管理员 token 访问超管 API | code `-400` |
| 3 | **水平越权** | 业务员 A 尝试查看业务员 B 的统计 | 只返回自己的数据 |
| 4 | **ID 遍历** | 业务员遍历 `/api/contact/1...` | 按脱敏规则返回或拒绝越权数据 |
| 5 | **批量枚举** | 循环调用 `/api/contact/search?q=1380...` | 触发频率限制 code `-402` |

```typescript
describe('越权攻击', () => {
  it('业务员不能访问其他业务员的统计', async () => {
    const _response = await app.request('/api/stat/me', {
      headers: { Authorization: `Bearer ${staff_b_token}` },
    })
    const _data = await _response.json()
    // 只返回 staff_b 自己的数据，不包含 staff_a 的
    expect(_data.data.user_id).toBe(staff_b_id)
  })
})
```

### D3. 注入攻击

| # | 攻击方式 | 测试方法 | 预期结果 |
|---|---------|---------|---------|
| 1 | **SQL 注入** | phone 参数传 `' OR 1=1 --` | 返回 400（校验拒绝） |
| 2 | **SQL 注入** | username 参数传 `admin'; DROP TABLE user; --` | 返回 400 |
| 3 | **XSS 注入** | 动态字段值传 `<script>alert(1)</script>` | 存储为纯文本，前端 HTML 转义 |
| 4 | **JSON 注入** | 请求体传超大 JSON（>1MB） | 返回 413 |
| 5 | **路径穿越** | URL 传 `/api/contact/../../api/field` | 返回 404 |

```typescript
describe('注入攻击', () => {
  it('SQL 注入应被 Zod 校验拦截', async () => {
    const _response = await app.request('/api/contact/search?q=' + encodeURIComponent("' OR 1=1 --"), {
      headers: { Authorization: `Bearer ${staff_token}` },
    })
    const _json = await _response.json() as any
    expect(_json.code).toBeLessThan(0)
  })

  it('超大请求体应被拒绝', async () => {
    const _huge_body = JSON.stringify({ data: 'x'.repeat(1024 * 1024) })
    const _response = await app.request('/api/contact/claim', {
      method: 'POST',
      headers: { Authorization: `Bearer ${staff_token}`, 'Content-Type': 'application/json' },
      body: _huge_body,
    })
    const _json = await _response.json() as any
    expect(_json.code).toBeLessThan(0)
  })
})
```

## E. E2E 测试（Playwright）

| # | 流程 | 步骤 |
|---|------|------|
| 1 | **业务员查询认领** | 登录 → 输入手机号 → 查看公海数据（明文） → 点击认领 → 强确认安全词校验 → 成功转入私海并挂上 90 天保护倒计时 |
| 2 | **私域资料撞单掩码** | 业务员 A 查询业务员 B 的已开发资源 → 界面关键信息展示 `123****6789`，并提示已被认领，无认领按钮 |
| 3 | **管理员批量导入** | 登录 → 上传 Excel → 查看阻挡防撞去重报告（检查是否正确标记了已开发的 `frozen_import`） → 输入强确认安全词执行 → 验证列表 |
| 4 | **个人与部门统计** | 业务员认领 3 个公海池号码 → 部门主管看板统计数据核对准确无误 |
| 5 | **90天掉落验证模拟** | 超管手动触发模拟过期策略 → 原 `developed` 号码重置回进入公海，前所有人信息抹除并允许再次被搜索、被任何人认领 |
| 6 | **动态字段管理** | 超管新增动态字段配置 → 批量导入包含新字段的数据 → 数据列表展示正确解析 JSON 数据 |
| 7 | **全局安全导出监控** | 超管执行下载全局联系人并触发导出 → 验证包含完整未脱敏数据 → 测试用普通管理账号调用立刻阻断（code -400） |
