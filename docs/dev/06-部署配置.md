# 部署配置

## 服务

- API：Cloudflare Workers，配置文件 `api/wrangler.jsonc`。
- 数据库：Cloudflare D1，迁移目录 `api/drizzle/`。
- 前端：Cloudflare Pages，构建产物 `web/dist`。

## API 配置

`api/wrangler.jsonc` 当前关键配置：

```json
{
  "name": "crm-api",
  "main": "src/index.ts",
  "compatibility_date": "2026-04-04",
  "d1_databases": [
    {
      "binding": "DB",
      "database_name": "crm-api",
      "migrations_dir": "drizzle"
    }
  ],
  "vars": {
    "ENVIRONMENT": "production",
    "FRONTEND_URL": "https://crmx.pages.dev"
  }
}
```

必须设置的密钥：

```bash
cd api
bunx wrangler secret put JWT_SECRET
```

可选运行时变量：

```bash
RATE_LIMIT_PER_MINUTE=120
RATE_LIMIT_WINDOW_SECONDS=60
FRONTEND_URL=https://crmx.pages.dev
```

## D1 初始化

```bash
cd api
bun install
bun run db:generate
bunx wrangler d1 migrations apply crm-api --remote
bunx wrangler d1 execute crm-api --remote --file src/seed.sql
```

本地开发：

```bash
cd api
bunx wrangler d1 migrations apply crm-api --local
bunx wrangler d1 execute crm-api --local --file src/seed.sql
bun run dev
```

默认管理员：

- 用户名：`admin`
- 密码：`Admin@123!`

上线后必须立即修改默认密码。

## 前端部署

```bash
cd web
bun install
PUBLIC_API_URL=https://<worker-domain>/ bun run build
```

Cloudflare Pages：

- 构建命令：`bun install && bun run build`
- 输出目录：`dist`
- 环境变量：`PUBLIC_API_URL=https://<worker-domain>/`

`web/public/_headers` 负责浏览器侧安全头。

## 生产 smoke

API 发布后执行：

```bash
API_BASE_URL=https://<worker-domain> bun scripts/smoke-production.mjs
```

如需验证登录态：

```bash
API_BASE_URL=https://<worker-domain> \
SMOKE_USERNAME=admin \
SMOKE_PASSWORD='Admin@123!' \
bun scripts/smoke-production.mjs
```

smoke 至少检查：

- API 根路径健康检查。
- D1 可用性。
- 可选登录。
- 可选 `/auth/check`。
- 可选基础受保护接口。
