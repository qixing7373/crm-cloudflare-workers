## 认证机制（JWT）

### Token 策略

```
登录成功 → 签发 access_token
         ├── 算法: HS256
         ├── 有效期: 24h
         ├── Payload: { userId, username, role, exp }
         └── 签名密钥: 环境变量 JWT_SECRET（≥32字符随机串）
```

### 密钥管理

```bash
# 生成密钥
openssl rand -base64 32

# 存储位置（不入代码库）
# 本地: .dev.vars 文件
# 生产: wrangler secret put JWT_SECRET
```

> ⚠️ JWT_SECRET 不可硬编码，不可提交到 Git。

### 密码复杂度校验

```typescript
function checkPassword(pwd: string): boolean {
  return (
    pwd.length >= 8 &&
    /[a-z]/.test(pwd) &&    // 小写字母
    /[A-Z]/.test(pwd) &&    // 大写字母
    /[0-9]/.test(pwd)       // 数字
  )
}
```

## 权限中间件

```typescript
// 角色层级: superadmin > manager > staff
const ROLE_LEVEL = { staff: 1, manager: 2, superadmin: 3 }

function requireRole(min_role: 'staff' | 'manager' | 'superadmin') {
  return async (c, next) => {
    const _user = c.get('user')
    if (!_user) return c.json({ code: -401, msg: 'Unauthorized', data: null })
    if (_user.status !== 'active') return c.json({ code: -103, msg: 'Account disabled', data: null })
    if (ROLE_LEVEL[_user.role] < ROLE_LEVEL[min_role])
      return c.json({ code: -400, msg: 'Forbidden', data: null })
    await next()
  }
}

// 导入权限（manager + superadmin + 授权的 staff）
// 使用
app.post('/api/import/sync', requireRole('manager'), syncImportHandler)
app.post('/api/field', requireRole('superadmin'), fieldHandler)
app.put('/api/user/:id/status', requireRole('manager'), statusHandler)
app.put('/api/user/:id/role', requireRole('superadmin'), roleHandler)
```

## 私海数据掩码中间件

> 查询返回时，如果客户处于他人私海，且查询人无穿透权限，必须对联系方式执行脱敏。

```typescript
// 脱敏级别判断（v4.0 双极模型）：
// 1. 超管 → 全部明文
// 2. 其他人查看他人已开发资源 → 强制打码

function maskPrivateData(
  contact: ContactRow,
  viewer: UserRow
): ContactRow {
  // 仅当资源已被他人认领时触发掩码
  const _is_others_private =
    contact.status === 'developed' &&
    contact.user_id !== viewer.id

  if (!_is_others_private) return contact  // 公海/自己的私海 → 明文

  // 超管 → 明文穿透
  if (viewer.role === 'superadmin') return contact

  // 其他所有情况 → 强制打码
  const _data = JSON.parse(contact.data)
  const _masked = { ...contact }
  _masked.phone = maskPhone(contact.phone)              // +668****5678
  _data.email = _data.email ? maskEmail(_data.email) : null
  _data.social = _data.social ? '***' : null
  _masked.data = JSON.stringify(_data)
  _masked._is_masked = true                              // 前端标识
  _masked._private_owner = _owner_username               // 提示"当前为张三私海"
  return _masked
}

function maskPhone(phone: string): string {
  if (!phone || phone.length < 6) return '***'
  return phone.slice(0, 4) + '****' + phone.slice(-4)
}
```

## 请求频率限制

当前代码提供基础 isolate 内存限流，并建议生产环境叠加 Cloudflare Rate Limiting Rules。

| 接口 | 限制 | 原因 |
|------|------|------|
| 路径 | 默认限制 | 说明 |
|------|------|------|
| `/auth/*` | `RATE_LIMIT_PER_MINUTE` | 登录、注册、校验、改密 |
| `/api/*` | `RATE_LIMIT_PER_MINUTE` | 所有业务接口 |

```typescript
app.use('/auth/*', rateLimit)
app.use('/api/*', rateLimit)
```

> 未配置 `RATE_LIMIT_PER_MINUTE` 时，代码限流不启用，便于测试和本地开发。

## 输入校验

所有 API 输入使用 **Zod** 校验：

```typescript
import { z } from 'zod'

const phoneSchema = z.object({
  phone: z.string().regex(/^\+?[1-9]\d{6,14}$/, '手机号格式不正确（支持国际号码）')
})

const registerSchema = z.object({
  username: z.string().min(3).max(20).regex(/^[a-zA-Z0-9_]+$/),
  password: z.string().min(8).refine(checkPassword, '密码需包含大小写字母和数字'),
})
```
