# 粉库系统 — i18n 多语言 & 错误处理

---

## i18n 方案

> 详细实现见 `09-前端方案.md` 中的 `vue-i18n.ts`

### 架构概览

```
src/vue-i18n.ts          # 插件初始化（懒加载 + Axios 联动）
src/i18n/zh-CN.ts        # 中文（默认内置，不懒加载）
src/i18n/en-US.ts        # 英文（按需懒加载）
```

### 核心特性

| 特性 | 说明 |
|------|------|
| 懒加载 | 默认只打包中文，英文 `import()` 动态加载 |
| Axios 联动 | 切换语言自动设置 `Accept-Language` 请求头 |
| HTML 联动 | 切换语言自动设置 `document.documentElement.lang` |
| Pinia 持久化 | 语言偏好持久化到 localStorage |
| meta 驱动 | 路由 `meta.title` 自动翻译为 `document.title` |

## 统一错误码系统

> **核心原则**：错误码是前后端唯一共享的契约。后端**一律返回 HTTP 200**，通过 JSON 中的 `code` 字段区分成功（`code > 0`）与失败（`code < 0`）。前端通过 `code` 查本地 i18n 字典翻译展示。前后端各自维护同一份错误码常量。

### API 响应结构

```typescript
// 成功
{ "code": 1, "msg": "success", "data": { ... } }

// 失败
{ "code": -101, "msg": "Invalid credentials", "data": null }
```

> `code > 0` 正常，`code < 0` 错误。`msg` 始终英文，仅作开发调试用。用户看到的提示由前端 i18n 翻译。

### 错误码字典（前后端共享）

| code | 模块 | msg (英文) | 中文翻译 (前端 i18n) |
|:----:|------|------------|---------------------|
| **-100** | Auth | `Username already exists` | 用户名已存在 |
| **-101** | Auth | `Invalid credentials` | 用户名或密码错误 |
| **-102** | Auth | `Account pending approval` | 账号审核中，请联系管理员 |
| **-103** | Auth | `Account disabled` | 账号已被禁用 |
| **-104** | Auth | `Token expired` | 登录已过期，请重新登录 |
| **-200** | Contact | `Phone number not found` | 该号码不在系统中 |
| **-201** | Contact | `Collision detected` | 该资源已被他人认领 |
| **-202** | Contact | `Contact is protected` | 已开发资源，禁止覆盖 |
| **-203** | Contact | `Confirm word mismatch` | 安全词输入错误 |
| **-204** | Contact | `Cannot edit locked field` | 该字段已锁定，无法修改 |
| **-300** | Import | `Unsupported file format` | 文件格式不支持 |
| **-301** | Import | `File too large` | 文件超过 20MB 限制 |
| **-302** | Import | `No phone column found` | 未检测到手机号列 |
| **-303** | Import | `Import token expired` | 导入令牌已过期，请重新上传 |
| **-400** | System | `Forbidden` | 权限不足 |
| **-401** | System | `Unauthorized` | 未登录或登录已失效 |
| **-402** | System | `Rate limited` | 请求过于频繁，请稍后再试 |
| **-500** | System | `Internal server error` | 系统异常，请稍后重试 |

```typescript
// src/errors/AppError.ts
import { ErrorMsg } from 'shared/codes'

export class AppError extends Error {
  constructor(public code: number) {
    super(ErrorMsg[code] || 'Unknown error')
  }
}

// 使用（无需指定 HTTP 状态码，一律返回 200）
throw new AppError(ErrorCode.COLLISION_DETECTED)
throw new AppError(ErrorCode.FORBIDDEN)
```

```typescript
// 全局异常处理中间件（HTTP 200 + code 区分）
app.onError((_error, c) => {
  if (_error instanceof AppError) {
    return c.json({ code: _error.code, msg: _error.message, data: null })
  }
  return c.json({ code: -500, msg: 'Internal server error', data: null })
})
```

### 前端实现

```typescript
// src/errors/codes.ts — 与后端保持完全一致（或从 shared/ 引入）
export { ErrorCode, ErrorMsg } from 'shared/codes'
```

```typescript
// src/api/http.ts — 全局响应拦截
// 后端一律返回 HTTP 200，通过 code 判断业务状态
http.interceptors.response.use(
  (res) => {
    const { code, msg } = res.data
    if (code < 0) {
      if (code === -104 || code === -401) {
        useUserStore().logout()
        window.location.href = '/login'
      }
      const _key = `error.${Math.abs(code)}`
      const _text = i18n.global().te(_key) ? i18n.t(_key) : msg
      message.error(_text)
      return Promise.reject(res.data)
    }
    return res.data
  },
  () => {
    // 仅在真正的网络错误（断网/超时）时触发
    message.error(i18n.t('error.500'))
    return Promise.reject()
  }
)
```
